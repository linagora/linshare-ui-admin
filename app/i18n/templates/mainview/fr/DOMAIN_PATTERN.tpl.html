Modèles de domaine

Après avoir créer une connexion à un annuaire de type LDAP, il est nécessaire de créer un modèle de domaine.

Ce modèle vous permettra de définir la correspondance entre les champs prénom, nom et email de {{ 'COMMON.NAME' | translate }}, et les attributs de votre annuaire. {{ 'COMMON.NAME' | translate }} nécessite 4 requêtes différentes pour fonctionner correctement :

  • requête LQL (Ldap Query Language) d'authentification : requête permettant de retrouver un utilisateur dans l'annuaire lors de la phase d'authenfication à partir de l'identifiant qu'il a saisi (couriel)
  • requête LQL (Ldap Query Language) utilisateur : requête permettant de retrouver un utilisateur dans l'annuaire lors des recherche pour consultation, édition, partage, ...
  • requête LQL (Ldap Query Language) d'autocomplétion sur tous les attributs : permet l'autocomplétion d'un utilisateur à partir du motif saisi
  • requête LQL (Ldap Query Language) d'autocomplétion sur le nom et prénom : permet l'autocomplétion d'un utilisateur à partir d'un couple de motif : nom-prénom ou prénom-nom
